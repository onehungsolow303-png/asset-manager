Topology Engine — Room Graph Structures

TopologyAgent generates an abstract graph before rooms are placed in space

Linear with Branches

Small dungeons, caves, mines. Clear progression.

Triggers: small_encounter size, linear map types (mine, maze)

Loop-Based

Large dungeons, castles. Multiple routes, shortcuts.

Triggers: large size, dungeon/castle/fort types

Hub-and-Spoke

Medium dungeons, temples, castles. Central hub with themed wings.

Triggers: medium/standard size, dungeon/temple/castle types

Hybrid (Hub + Loops)

Large dungeons. Hub provides orientation, wings have internal loops.

Triggers: large/open_world size, or explicit hybrid profile

How TopologyAgent Works

Input: map type profile, map size, seed, room count estimate
Output: abstract RoomGraph with nodes (room slots) and edges (required connections)

Steps:
1. Select topology type from profile preference list (size can override)
2. Generate base graph structure (spanning tree for linear, cycle insertion for loops, star graph for hub)
3. Assign zone IDs to nodes (outer→middle→inner→boss progression)
4. Tag special nodes: entrance (always zone 0), boss (always deepest zone), key rooms
5. Insert optional edges: shortcuts (one-way), secret passages, locked doors
6. Annotate edges with connection type: corridor, door, locked_door, secret, one_way, stairs
